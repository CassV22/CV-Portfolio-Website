The Daily Redesign

arrow_back

Back

This is the redesign of a website for a pre-college summer camp
that I worked at. The site was intended to be used by students
to check and sign up for daily activities, but many chose not to used
it beacuse of its poor design and inconvience. Using the UX design process,
I recreated the site in a way that both better fit the needs of the intended
users and expanded its utility.

The original website for the program, which was originally created using
Google Sites. UX research showed that the main user pain points with this
site were:
1) poor layout/navigation
2)lack of visuals/color
3)lack of information.
Because of these frustrations, targeted users avoided using the site, which
led to more work and frustrations for the program staff.

This is the first redesign of website done using Google Sites. Major design
changes included the addition of more relevant and detailed information
based on UX research, use of pictures that represent the program and offered activities, and use of colors that liven up the website while also
better reflecting the brand. To overcome some of the limitations of Google
Sites and to make a more dynamic and engaging site, the final version was
coded from scratch using HTML, CSS, and JavaScript. A walk-thru of the
site can be viewed below.
